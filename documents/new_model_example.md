# 시장 지표들을 활용한 리밸런싱 주기 단축 모델

## **Objective & Problem Definition**
- 기존 TP(TFT+Pessimistic)나 TOTAL(TFT+Equal Weight) 알고리즘 모두 재무제표 기반의 모델이기 때문에 분기별로 포트폴리오 추출 및 리밸런싱 진행
- 중장기적으로 해당 기업의 성장성을 판단해서 투자를 한다는 측면에서는 매우 합리적
- 하지만 주식 시장은 매우 변동성이 크고, 미래에 대한 예측이 어려운데 3개월이라는 긴 시간 동안 최대손실률 기반의 수시 리밸런싱만에 의존해서 리스크 관리를 하기에는 어려움이 존재
- 따라서 현 알고리즘의 정기 리밸런싱 주기를 줄이고, 시장 위험에 발빠르게 대처하기 위한 새로운 알고리즘 개발 필요
---
## **Background / Motivation**
- 일반적으로 연구되는 다양한 해외 시장의 AI 모델들은 단기 예측 (하루, 일주일 등)에 특화되어 있음
    - 실제 주식시장에서는 매매 수수료 등의 문제로 단기 예측 모델 사용에 어려움이 존재
- 기존 재무제표 기반 모델의 경우 데이터가 3개월마다 업데이트되고, 그에 따라 라벨도 3개월 간격이기 때문에 현 상황에서 이를 단순히 줄이는 것은 불가능함
- 금융 시장의 지표들 중에서 더 짧은 단위 (e.g. 2주, 1개월 등)로 공시가 되거나 업데이트 되는 지표가 존재하지 않음
---
## **Strengths & Advantages**
1. **(1) 성장성 팩터 예측 (2) 성장성 높은 종목 기반 포트폴리오 구성** 이라는 기존 모델은 성장성 팩터가 실제 높아도 주가의 상승으로 이어지지 않을 수 있다는 단점이 존재하는 반면, 이 모델은 라벨이 실제 주가 기반으로 만들어지기 때문에 예측에 성공한다면 수익 보장
2. 수익률 기반이기 때문에 리밸런싱 주기를 좀 더 유연하게 설정가능함
    - 라벨 설정을 어떻게 하냐에 따라서 리밸런싱 주기를 다르게 할 수 있음
3. 충분한 데이터 확보가 가능함
    - 기존 모델의 경우 dart에서 공시되는 보고서에 의존하고, dart api의 경우 2015년 4분기 보고서부터 제공을 하기 때문에 데이터 확보에 어려움이 있었음
    - krx 정보데이터시스템에서 제공하는 가격 데이터 (OHLCV + 거래대금, mktcap), PER, PBR 등은 기본적으로 일별 데이터이기 때문에 데이터의 수 자체가 많고, 2000년정도부터 제공하기 때문에 과거 다양한 시장 상황에 대한 백테스팅을 진행할 수 있음
        - 다양한 시장 상황을 고려한 파라미터 세팅이 가능함
---
## **Weakness**
1. 일별 데이터 기반으로 만든 라벨이고, 기존 라벨보다 주기가 더 짧음
→ 높은 변동성으로 인해 예측의 난이도가 더 높음
2. 데이터가 많아지는만큼 학습에 시간이 오래 걸릴 수 있음
    - 현재 모델처럼 cpu 환경에서 학습시키기에 무리가 있을 것으로 보임
    → UOS의 슈퍼컴퓨팅 자원을 활용하면 해결 가능
3. 만약 기존 재무제표 데이터를 같이 사용한다면 데이터 증진의 효과를 얻기는 어려움
    - 리밸런싱 주기 단축은 가능 (리밸런싱 주기는 결국 라벨 주기에 의존하기 때문)
---
## **Data Description**
- **Label**: 수익률 (2주 ~ 1개월로 설정)
- **Input**: 가격 데이터, 재무 데이터 등을 모두 사용
   1. 2000년부터 수집해서 백테스팅 기간 늘리기
   2. 코스피 + 코스닥 전 종목에 대한 가격 데이터 (일별)
   3. PER, PBR 등의 재무 데이터 (일별)
   4. 기업 공시 보고서 기반 재무제표 데이터 (분기별)
   (2015년 4분기부터 수집 가능)
   5. 시장 지표
   6. 기술적 지표 or 팩터 (Alpha 158, Alpha 360 참고)
---

## **Model Architecture Description**
- **Arichitecture**
   - Trasnformer 기반 시계열 예측 모델 (Supervised Learning Model)
   - Backbone: `MASTER`
- **Loss function**
   - MSE or Ranking Loss (결합도 가능)
       - MSE만을 사용 → 예측값을 얻은 후 어떻게 처리할건지에 대한 추가적인 로직 필요
       - Ranking Loss만을 사용 → Top-K Equal Weight Portfolio 구성 가능

## **Additional Considerations**
- 기존 dart 공시 기반 재무제표를 통한 TFT 예측 결과와 이번에 새로 구축하는 모델의 예측 결과를 모두 활용 (intersection을 유니버스로 간주하는 방식)
- 지도학습 기반 말고, SR을 높이는 방식의 강화학습 모델도 고려중
    - 강화학습의 경우 지식이 부족하여 추가적인 공부가 필요 (learning curve가 매우 길어진다는 단점 존재)
- 이 모델도 Top-K 선정, 양수인 기업 선정 등을 통해 유니버스 축소에만 집중하고, 최종 포트폴리오 구축시에는 Pessimistic 기반의 리스크 최소화 알고리즘을 적용하는 것도 가능
- 각 종목의 관계 모델링을 수행할 것인지
    - 이렇게 했을 때 발생하는 문제점은 구성종목을 고정해야함
        - 종목과 수가 변하면 그에 대응되는 weight를 구성하기가 어려움
    - 기존 모델은 구성종목의 관계를 고려하지 않고, 오직 시계열 패턴만을 분석하는 단일 종목 예측 모델이라고 볼 수 있음 (학습시에는 모든 종목의 패턴을 종합적으로 고려하긴 하지만 관계를 직접적으로 모델링하지는 않음) (종목 차원을 그냥 배치로 간주했다고 볼 수 있음)
- 학습 대상 유니버스를 어떻게 설정할 것인지
    - 생존편향(Survivorship Bias): 과거 데이터를 분석할 때 현재까지 '살아남은' 종목들만을 대상으로 하는 오류를 의미함. 이는 특히 장기 백테스팅에서 심각한 문제를 야기할 수 있음
    - 미래예측편향(Look-Ahead Bias): 전략을 수립하거나 의사결정을 할 때 아직 알 수 없는 미래 정보를 사용하는 오류를 의미함. 이는 백테스팅의 가장 큰 함정 중 하나임.
